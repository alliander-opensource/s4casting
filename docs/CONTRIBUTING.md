# CONTRIBUTING

Thank you for your interest in contributing to **s4casting**!

## Development Workflow

- Ensure your code passes linting and all tests before submitting by running `make check` in the root directory.

## Pull Requests

- Create a new branch for your feature or fix.
- Write a clear description of your changes.
- Ensure all tests pass (`make check`).

## Reporting Issues

- Use GitHub Issues for bugs and feature requests.
